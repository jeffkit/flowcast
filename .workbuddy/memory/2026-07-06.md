- **核心新发现（GAP-ANALYSIS 未覆盖）**：L3 生成 flow 的 import 白名单（`orchestrator/validate.js` 的 `scanImports`）只堵模块通道，未堵 Node 全局对象通道（`process.env` / `fetch` / `eval` / `new Function` / `globalThis`），生成的 flow 可外泄密钥或外联网络。
- **更严重的不对称**：dry-run（`validate.js`）用最小 env（假 HOME、无密钥），但真实执行路径 `subflow.js runFlow` 是 `const env = {...process.env}` 完整继承宿主 env，dry-run 的 hardening 在真实执行时被一笔勾销。
- 还有动态命名空间 bypass：`import * as fc from 'flowcast'; fc.spawnCapture()` 无法静态拦截（代码注释自认）。
- 建议：① `runFlow` 真实执行收窄 env；② `scanImports` 增补全局对象扫描；③ 明示 L3 信任模型；④ 动态命名空间改 AST 扫描或明文禁止。
